# Metrics

The North Star metric is qualified savings rupees discovered per week. This matches the product's job better than DAU because founders will not use an AI spend audit daily; they use it when bills change, budgets tighten, or procurement starts. Rupee savings discovered also aligns Credex and the user: if the audit finds real avoidable spend, the user gets value and Credex gets a relevant lead.

Three input metrics drive it. First, completed audits, because no completion means no value. Second, percentage of audits with over ₹50,000/month potential savings, because those are the Credex-ready opportunities. Third, report capture rate after result, because email capture after value is the bridge from free tool to sales conversation.

I would instrument the form funnel first: page view, first tool added, audit submitted, audit completed, lead captured, share link copied, public report viewed, and consultation clicked. I would also log recommendation categories so Credex can see whether demand is mostly API credits, plan downgrades, or tool consolidation.

A pivot trigger: after 500 completed audits from the target audience, if fewer than 5% show over ₹50,000/month savings or fewer than 10% of high-savings users leave an email, the tool is either attracting the wrong audience or not finding painful enough problems. The next pivot would be invoice upload or API-key billing import to reduce self-reported data errors.
